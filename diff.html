üìò Java String vs StringBuffer ‚Äì Common Methods & Examples

1. toString()

// String
String s = "Hello";
System.out.println(s.toString()); // Output: Hello

// StringBuffer
StringBuffer sb = new StringBuffer("Hello");
System.out.println(sb.toString()); // Output: Hello

2. append()

// String (creates new object)
String s = "Hello";
s = s + " World";
System.out.println(s); // Hello World

// StringBuffer (in-place)
StringBuffer sb = new StringBuffer("Hello");
sb.append(" World");
System.out.println(sb); // Hello World

3. insert()

// StringBuffer only
StringBuffer sb = new StringBuffer("Hello");
sb.insert(5, ",");
System.out.println(sb); // Hello,

4. delete()

// StringBuffer only
StringBuffer sb = new StringBuffer("Hello, World");
sb.delete(5, 6);
System.out.println(sb); // Hello World

5. reverse()

// StringBuffer
StringBuffer sb = new StringBuffer("Java");
sb.reverse();
System.out.println(sb); // avaJ

6. length()

// String
String s = "Data";
System.out.println(s.length()); // 4

// StringBuffer
StringBuffer sb = new StringBuffer("Data");
System.out.println(sb.length()); // 4

7. charAt()

// String
String s = "World";
System.out.println(s.charAt(1)); // o

// StringBuffer
StringBuffer sb = new StringBuffer("World");
System.out.println(sb.charAt(1)); // o

8. setCharAt() (StringBuffer only)

StringBuffer sb = new StringBuffer("Test");
sb.setCharAt(0, 'B');
System.out.println(sb); // Best

9. replace()

// String
String s = "abcabc";
s = s.replace("a", "x");
System.out.println(s); // xbcxbc

// StringBuffer
StringBuffer sb = new StringBuffer("abcabc");
sb.replace(0, 3, "XYZ");
System.out.println(sb); // XYZabc

üîÅ Major Differences

  • String is immutable: operations return new objects.
  • StringBuffer is mutable: changes happen in-place.
  • StringBuffer is thread-safe (synchronized), unlike StringBuilder.
